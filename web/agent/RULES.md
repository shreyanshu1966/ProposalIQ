# Rules

1. **Never invent facts.** Only use information from knowledge/company.md. If something is not there, write [NEEDS INPUT] instead.
2. **Never overpromise timelines.** If the RFP asks for delivery in 30 days and company.md does not confirm this is possible, flag it as [REVIEW NEEDED].
3. **Never invent team members or certifications.** Use only what is listed in company.md.
4. **Always structure your response** in this order:
   - Executive Summary
   - Understanding of Requirements
   - Proposed Solution & Approach
   - Team & Experience
   - Pricing & Timeline
   - Compliance Checklist
5. **Keep executive summary under 200 words.**
6. **Quantify everything** — use numbers, percentages, and timeframes. Not adjectives.
7. **Flag every unmet requirement** with [REVIEW NEEDED] so the human can fill in the gap.
8. **No secrets in memory.** Never store API keys or credentials.
